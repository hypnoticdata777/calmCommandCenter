## 3. CSS Variable Architecture

**What we added to `theme.css`:**

Three new variables added to both `:root` (light) and `.dark` (dark):

| Variable | Light value | Dark value | Purpose |
|---|---|---|---|
| `--brand-color` | `#0e7490` | `#dc2626` | Brand accent color |
| `--blob-rgb` | `14, 116, 144` | `220, 38, 38` | RGB channels for use inside `rgba()` |
| `--blob-tint` | `#040f11` | `#1a0505` | Ambient overlay tint on background animations |

`--color-brand: var(--brand-color)` was registered in `@theme inline` to expose Tailwind classes `text-brand`, `bg-brand`, `border-brand`.

**Why `--blob-rgb` as separate channels:**
CSS `var()` cannot be used inside `rgba(var(--color), 0.4)` if the variable contains the full `rgb(r,g,b)` notation. Storing just the channels (`14, 116, 144`) allows `rgba(var(--blob-rgb), 0.4)` to work correctly in inline styles and CSS string values.

## 4. ThemeProvider Wiring — main.tsx

**What changed:**
Imported `ThemeProvider` from `next-themes` and wrapped the full app tree.

```
ThemeProvider → BrowserRouter → AppRouter → Layout → Page
```

**Config decisions:**
- `attribute="class"` — adds `class="dark"` to `<html>`, which is what `@custom-variant dark` expects
- `defaultTheme="dark"` — app boots in dark mode (original design preserved)
- `enableSystem={false}` — does NOT auto-detect OS dark/light preference; user controls it manually
- `storageKey="h777-theme"` — saves choice in localStorage under this key

## 5. Theme Toggle — Navbar.tsx

**What changed:**
Added `useTheme()` hook from `next-themes` and `Sun`/`Moon` icons from `lucide-react`.

**Mounted guard pattern:**
```tsx
const [mounted, setMounted] = useState(false);
useEffect(() => setMounted(true), []);
const ThemeIcon = mounted ? (theme === "dark" ? Sun : Moon) : null;
```
This prevents a hydration flash — next-themes reads localStorage only on the client, so we wait until the component is mounted before rendering the correct icon.

**Toggle placement:**
| Nav state | Toggle location |
|---|---|
| Desktop top bar | Right side, after nav links |
| Desktop side pill | Bottom of pill |
| Mobile overlay | Top-left corner, mirroring the X close button |

**All hardcoded colors updated:**
- `text-white/X` → `text-foreground/X`
- `bg-white/5` → `bg-foreground/5`
- `border-white/10` → `border-foreground/10`
- `text-[#dc2626]` (active links) → `text-brand`
- `bg-[#0a0a0a]/96` (mobile overlay) → `bg-background/96`

## 7. Animated Background Updates

### BeatingBrain.tsx
All four blob `radial-gradient` values updated to use `rgba(var(--blob-rgb), X)` instead of hardcoded `rgba(220, 38, 38, X)`. The ambient tint overlay changed from `bg-[#1a0505]` to `style={{ backgroundColor: "var(--blob-tint)" }}`.

Result: In light mode, blobs glow **cyan**. In dark mode, they glow **red** (original).

### RollingHeptagon.tsx
- SVG `drop-shadow` filter updated to `rgba(var(--blob-rgb), 0.3)`
- Polygon `stroke` moved from a JSX attribute to a CSS `style` property so `rgba(var(--blob-rgb), ${opacity})` resolves correctly
- Rolling dot changed from `bg-[#dc2626]` to `bg-brand`

Result: Heptagon outline and inner dot shift from red to cyan in light mode.

```
main.tsx
  └── ThemeProvider (next-themes, defaultTheme="dark", storageKey="h777-theme")
        └── BrowserRouter
              └── AppRouter
                    └── Layout  (Navbar persists across all pages)
                          ├── /           → App.tsx (Home)
                          ├── /work       → Work.tsx
                          ├── /lab        → Lab.tsx
                          ├── /journal    → Journal.tsx

                          └── /contact    → Contact.tsx
```

**Theme flow:**
```
ThemeProvider adds class="dark" to <html>
  → @custom-variant dark in theme.css activates .dark variables
  → --brand-color, --background, --foreground etc. swap values
  → All components using text-brand, bg-background, text-foreground update automatically
```
