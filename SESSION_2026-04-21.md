## 1. Comment Style Refactor — App.tsx

**What we did:**
Rewrote all comments in `src/app/App.tsx` from a mixed/Spanish style to a strict 4-layer English format.

**The 4 layers:**
1. **Tags table** — bordered ASCII two-column table at the top listing every concept used and its purpose
2. **Page description block** — "What the user sees" and "Structure" explanation
3. **Section banner comments** — `── NAME ──────` dashes above every major block
4. **Inline `//` comments** — short one-liners explaining the WHY behind non-obvious lines

**Files changed:**
- `src/app/App.tsx` — full comment rewrite
- `src/styles/theme.css` — added `.font-display` utility class to replace inline `style={{ fontFamily }}` on principle number divs (removed 3 linter warnings)

## 2. Git Conflict Resolution

**What happened:**
After committing the App.tsx refactor, `git push origin main` was rejected because the remote had commits we didn't have locally. Attempts to `git pull --rebase` hit merge conflicts in App.tsx. The stash pop also failed mid-rebase.

**How we fixed it:**
```
git rebase --abort      # exit the broken rebase cleanly
git stash pop           # restore stashed changes
git pull origin main --no-rebase   # merge instead of rebase
# resolved conflict in VSCode by clicking "Accept Current Change"
git add src/app/App.tsx
git commit -m "merge: resolve conflict in App.tsx"
git push origin main
```

**Lesson:** When a rebase conflicts mid-way, always abort first before trying anything else.

## 4. Color Palette Documentation

Identified and documented every color in use on the site:

| Token | Value | Usage |
|---|---|---|
| Background | `#0a0a0a` | Page background |
| Accent | `#dc2626` | Headings, active links, buttons |
| Text full | `white` | Primary headings |
| Text 90% | `white/90` | Body text |
| Text 80% | `white/80` | Manifesto lines |
| Text 70% | `white/70` | Principle descriptions |
| Text 60% | `white/60` | Tagline, subtle copy |
| Text 40% | `white/40` | Form label |
| Text 30% | `white/30` | Input placeholder |
| Dividers | `white/10` | Horizontal rules, borders |
| Input bg | `white/5` | Input field background |

**Rule established:** All new pages and components must use only these tokens. No new colors added without deliberate decision.

## 5. Navigation Architecture Decision

**Agreed page structure:**

| Route | Name | Purpose |
|---|---|---|
| `/` | Home | Hero, principles, email capture |
| `/work` | Work | Selected projects |
| `/lab` | Lab | Open source tools + experiments |
| `/journal` | Journal | Writing on systems and ops |
| `/about` | About | Who's behind this |
| `/contact` | Contact | Direct contact |

**Nav behavior agreed:**
- Desktop at top of page → transparent horizontal top bar
- Desktop on scroll → slides to right side as vertical floating pill, fades to 35% opacity, reappears fully on hover
- Mobile/tablet → nav completely hidden, small `☰` button in top-right corner, tapping opens full-screen dark overlay with staggered link animation

## 9. Architecture After This Session

```
main.tsx
  └── BrowserRouter
        └── AppRouter
              └── Layout  (Navbar persists across all pages)
                    ├── /           → App.tsx (Home)
                    ├── /work       → Work.tsx
                    ├── /lab        → Lab.tsx
                    ├── /journal    → Journal.tsx
                    ├── /about      → About.tsx
                    └── /contact    → Contact.tsx
```

**Key principle:** `App.tsx` was never broken. It remained the home page component — we wrapped around it, never inside it.
